1 Gather Info from Apple's Website

Create an App Record in App Store Connect Website

Go to appstoreconnect.apple.com → My Apps → click + → New App. Fill in:

Platform: iOS Name: SayFit Primary Language: English (U.S.) Bundle ID: com.sayfit.app SKU: sayfit-app-001 (any unique string, doesn't matter)
After creating, look at the URL — it will contain a number like appstoreconnect.apple.com/apps/1234567890. Write that number down — you'll need it in Step 4.
Add a Privacy Policy URL Website

In App Store Connect → Your App → App Information, paste a URL into the Privacy Policy field. Apple requires this before you can submit.

Don't have one? Go to privacypolicygenerator.info, fill out the form, copy the text into a Google Doc, turn on link sharing, and paste that URL here. Takes 5 min.
Find your Apple Team ID Website

Go to developer.apple.com → Account → Membership Details. Your Team ID is a 10-character string like A1B2C3D4E5. Write it down — needed in Step 4.

2 Get Your Supabase Keys

Copy your project keys from Supabase Website

Go to supabase.com → your project → Settings → API. Copy both values and save them — you'll paste them into the terminal in Step 3.

Project URL: https://xxxxxxxxxxxx.supabase.co anon public: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... (long string)

3 Terminal — One-Time Setup

Navigate to the project folder Terminal
cd /Users/jackbowker/sayfit-app

Run every terminal command in this folder.

Install EAS CLI and log in Terminal
npm install -g eas-cli eas login

Log in with your Expo account. Create a free one at expo.dev if you don't have one.

Link the project to your Expo account Terminal
eas init

This automatically fills in the Project ID in app.json. Say yes to any prompts.

Add your Supabase keys as build secrets Terminal

Paste your actual values from Step 2 below (replace the placeholder text):

eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_URL --value "https://your-project.supabase.co" eas secret:create --scope project --name EXPO_PUBLIC_SUPABASE_ANON_KEY --value "eyJ..."

4 Fill In Your Credentials File

Open eas.json and fill in your three values File

Open eas.json in the project folder. Find the submit section and replace the empty strings:

"submit": { "production": { "ios": { "appleId": "you@icloud.com", ← your Apple ID email address "ascAppId": "1234567890", ← the number from App Store Connect URL (Step 1) "appleTeamId": "A1B2C3D4E5" ← from developer.apple.com Membership (Step 1) } } }

Save the file when done.

5 (Optional) Test Voice Input Before Submitting

Build a development client to test the mic on your phone Optional Terminal

The voice mic button requires native iOS code and won't work in Expo Go. Skip this if you're happy to test it in TestFlight. Do this if you want to test it now.

eas build --platform ios --profile development
EAS will give you a QR code / link when it's done (~15 min). Scan it on your phone to install the dev app. Then start your dev server with:

npx expo start --dev-client

Open the installed app (not Expo Go) and scan the QR code. The mic button will now work.

6 Build the Production App

Run the production build Terminal

This sends your code to Expo's servers and compiles the iOS app. Takes about 15–20 minutes. You can close the terminal — you'll get an email when it's done.

eas build --platform ios --profile production
EAS will ask for your Apple ID password and may send a 2FA code to your phone. It handles certificates and provisioning profiles automatically — just approve what it asks.

7 Submit to TestFlight

Submit the build to Apple Terminal

Once the build finishes, run this to send it to TestFlight:

eas submit --platform ios --profile production
The build appears in App Store Connect → TestFlight within a few minutes. Apple's automated review takes 15–30 min, after which you can invite testers.

8 Invite Testers

Add internal testers in App Store Connect Website

Go to appstoreconnect.apple.com → Your App → TestFlight → Internal Testing → Add Testers. Enter each tester's Apple ID email. They'll get an invite and install via the TestFlight app.

Internal testers (up to 100 people) get the build immediately with no extra Apple review. External testers (public link) require a short extra review — usually same day.
